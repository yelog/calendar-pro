# 设置窗口色彩平衡设计

**日期：** 2026-04-01

**目标：** 降低设置窗口左右分栏的色差，修正标题与内容区边距不一致的问题，让界面更接近 macOS 原生设置窗口的低对比双栏风格。

## 现状问题

1. [SettingsRootView.swift](/Users/yelog/workspace/swift/calendar-pro/CalendarPro/Views/Settings/SettingsRootView.swift) 中左栏整块使用 `underPageBackgroundColor`，右栏使用 `windowBackgroundColor`，同一窗口被切成两块色温明显不同的区域。
2. 左右栏之间既有背景反差，又有单独 `Divider()`，边界过硬。
3. 左侧未选中导航项也保留描边，进一步增加了密度和灰度。
4. 右侧各页的内容边距不一致，`About` 为 30pt，而其他页面多为 20pt 或 24pt，标题与正文没有稳定对齐。

## 方案比较

### 方案 A：低对比原生双栏（推荐）

- 统一窗口基底为 `windowBackgroundColor`
- 侧边栏不再单独整块染灰，只保留一条弱分隔线
- 通过选中态卡片和图标圆底表达导航层级
- 统一右侧内容水平边距

**优点：**
- 最接近 macOS 设置窗口的观感
- 可以直接消除当前“左重右轻”的断层
- 改动集中在视图容器层，风险低

**缺点：**
- 需要顺手微调部分按钮和卡片的对比度

### 方案 B：保留灰色侧边栏，仅降低透明度

- 继续保留左栏独立底色
- 只调整透明度和分割线强度

**优点：**
- 改动最小

**缺点：**
- 左右仍然是两个色块，问题只会缓和，不会根治

### 方案 C：右侧重做成更强的卡片化详情页

- 加大右侧卡片与分组层级
- 用更强内容容器对冲左侧底色

**优点：**
- 层级明确

**缺点：**
- 会把这次问题演变成整套设置页重绘，超出当前需求

## 选定方案

采用方案 A。

## 设计要点

1. 根容器统一为 `windowBackgroundColor`，避免左深右浅。
2. 侧边栏通过弱分隔线定义区域，不再依赖整块底色。
3. 未选中导航项取消常驻描边，仅保留选中态高亮卡片。
4. 右侧标题区与各设置页正文统一为 30pt 水平边距。
5. 摘要卡片与分隔线适度减弱，避免右侧因为容器太白而再次失衡。

## 验证方式

1. 打开设置窗口，确认左右区域不再出现明显色块断层。
2. 切换导航项，确认选中态依然清晰。
3. 检查“菜单栏”“日程”“地区与节假日”“关于”页的标题与正文左边界是否对齐。
4. 构建项目，确认没有引入编译回归。
