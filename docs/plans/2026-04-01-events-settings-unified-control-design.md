# 日程设置统一控制设计

## 背景

当前“日程”设置页把“日历日程”和“提醒事项”拆成两个并列区块，并分别使用 `showEvents` 与 `showReminders` 控制。实际运行时，`showEvents` 已经承担了整个面板日程模块的总开关职责，而 `showReminders` 只是该模块下的一个数据源开关。现有 UI 把两者摆成并列层级，容易造成语义错位：

- 用户会把“显示日程”理解为“显示日历日程”
- 但代码里它实际上控制整个面板是否加载任何日程数据
- 提醒事项被展示成一级功能，和实际归属关系不一致

本次调整的目标是让设置页的层级与运行逻辑一致，同时保留现有来源筛选能力。

## 目标

- 提供一个真正清晰的总开关，统一控制面板中的日程模块
- 将“日历日程”和“提醒事项”明确为总开关之下的两个数据源
- 关闭总开关时保留用户已选的数据源与来源列表
- 保持旧配置升级后的行为稳定，不让现有用户的设置突然变化

## 非目标

- 不改动面板中的日程列表展示样式
- 不合并日历与提醒事项的来源选择模型
- 不在本次调整中重做权限申请流程

## 信息架构

设置页中的“日程”只保留一个主区块，内部结构如下：

1. 主开关：`在面板中显示日程`
2. 次级说明：关闭后不显示日历日程与提醒事项，但会保留已选来源
3. 当主开关开启时，显示“数据来源”区域
4. 数据来源区域包含两个子开关：
   - `包含日历日程`
   - `包含提醒事项`
5. 子开关开启后，才展示各自的来源列表

这种结构把“是否显示整个模块”和“模块里包含哪些来源”拆成两层，符合用户认知，也贴合 macOS 设置页的层级表达。

## 交互设计

### 总开关

- 关闭总开关时，面板不加载任何日程或提醒事项数据
- 关闭总开关不会改写两个子开关状态，也不会清空来源列表选择
- 再次打开总开关时，恢复用户此前的子开关组合与来源选择

### 子开关

- `包含日历日程` 控制日历事件数据源
- `包含提醒事项` 控制提醒事项数据源
- 子开关关闭时，仅折叠来源列表并停止对应数据源加载
- 子开关关闭不会清空已选来源列表

### 权限与空状态

- 没有权限时，在对应子区块下显示紧凑权限提示和请求按钮
- 有权限但没有数据时，显示“暂无可用日历”或“暂无可用提醒事项列表”
- 总开关开启但两个子开关都关闭时，面板显示“未启用任何日程来源”

## 状态模型

现有模型：

- `showEvents`
- `enabledCalendarIDs`
- `showReminders`
- `enabledReminderCalendarIDs`

调整后：

- `showEvents`：继续承担整个日程模块的总开关职责
- 新增 `showCalendarEvents`：控制是否包含日历日程
- `showReminders`：继续控制是否包含提醒事项
- `enabledCalendarIDs`：日历来源过滤
- `enabledReminderCalendarIDs`：提醒事项来源过滤

这样能最小化对现有数据结构和业务逻辑的扰动，同时让字段语义与 UI 结构对齐。

## 兼容性与迁移

`MenuBarPreferences` 当前通过 `Codable` 直接解码旧配置。新增字段后必须保证旧版本持久化数据仍能正常加载。

迁移策略：

- 旧配置中的 `showEvents` 直接作为新的总开关值
- 新增的 `showCalendarEvents` 在旧配置缺失时，默认回落到 `showEvents`

结果是：升级前如果用户开启了日程模块，升级后默认仍会显示日历与提醒事项；不会因为新增子开关导致日历来源被意外关闭。

## 运行逻辑调整

面板数据加载逻辑改为：

- `showEvents == false`：直接清空并停止加载
- `showEvents == true && showCalendarEvents == true`：加载日历事件
- `showEvents == true && showReminders == true`：加载提醒事项
- 两个子开关都为 `false`：返回空结果并显示“未启用任何日程来源”

## 文案更新

设置页摘要从当前的简单布尔拼接，调整为更符合层级的表达：

- `日程已关闭`
- `日历开 / 提醒开`
- `日历开 / 提醒关`
- `日历关 / 提醒开`
- `未启用任何日程来源`

## 测试策略

需要覆盖以下场景：

- 旧版偏好数据可解码，新字段使用迁移默认值
- 总开关关闭时，不加载日历也不加载提醒事项
- 仅开启日历子开关时，只返回日历事件
- 仅开启提醒事项子开关时，只返回提醒事项
- 关闭总开关后重新开启，来源列表选择仍然保留
- 设置页摘要文案随新状态正确变化
