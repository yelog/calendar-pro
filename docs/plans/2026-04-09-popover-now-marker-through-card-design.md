# 当前时间线横穿进行中日程设计

**日期：** 2026-04-09

**目标：** 让下拉日程区域中的当前时间红线在进行中日程场景下直接横穿当前卡片，并绘制在卡片内容上层，视觉上更接近 Apple Calendar。

## 背景

2026-04-02 的当前时间 marker 设计把红线停在卡片左缘，只用一个短入口刻度表达“当前时间已进入卡片”。这种分层方案能避免正文被遮挡，但不符合这次目标中的参考体验：用户更希望红线像 Apple Calendar 那样直接穿过当前日程，并明确悬浮在卡片上方。

这次变更只调整 `withinItem` 场景的视觉表达，不改变当前时间 marker 的业务判定、滚动定位和时间进度计算逻辑。

## 设计目标

1. 当前时间红线在进行中日程场景下应横穿整张卡片。
2. 红线必须位于卡片内容上层，不能被卡片背景或边框截断。
3. 红点、时间胶囊和横线继续共用同一条 Y 坐标，保留“当前进度落点”语义。
4. `beforeGroup` / `afterGroup` 的 marker 行为保持不变，避免影响非进行中场景。

## 非目标

1. 不修改 `beforeGroup` / `afterGroup` 的布局语言。
2. 不重做 `EventCardView` 的内容结构和点击行为。
3. 不改变 `EventTimelineSnapshot`、`CalendarItem.timelineProgress` 等业务逻辑。

## 方案对比

### 方案 A：继续停在线卡片左缘，只优化入口刻度

- 优点：改动最小
- 缺点：仍然不满足“整条横穿当前日程”的目标

### 方案 B：在卡片内部预留穿线通道

- 优点：更容易控制文字可读性
- 缺点：视觉上仍然像“给线让位”，不够接近 Apple Calendar

### 方案 C：全局 overlay 直接横穿卡片

- 做法：保留 `EventListView` 的全局 overlay，在 `withinItemMarkerOverlay` 中将红线从时间轴一直绘制到当前卡片右侧附近，并删除卡片左缘短刻度
- 优点：最直接满足目标，改动集中，能复用现有 marker 坐标体系
- 缺点：需要控制透明度和终点 inset，避免过度压住正文和 disclosure

## 选定方案

采用方案 C。

## UI/UX 规则

### 进行中日程

1. 当前时间胶囊仍显示在时间文本轨中。
2. 当前时间红点仍显示在时间轴轨中。
3. 横向红线从时间轴中心开始，直接穿过当前卡片，到达卡片右边界内侧。
4. 红线继续浮在卡片内容上层。
5. 原有的卡片左缘短入口刻度移除，避免与整条红线重复表达。

### 可读性控制

1. 红线维持细线厚度，不加粗成高亮条。
2. 红线颜色保留当前红色语义，但透明度略低于红点，避免过度压住标题和时间文本。
3. 红线纵向位置继续由 `progress` 决定，不改成固定居中。

## 技术设计

1. 保持 `EventTimelineMarkerPosition.withinItem(selectionIdentifier:progress:)` 不变。
2. 继续通过 `EventTimelineItemBoundsPreferenceKey` 获取进行中卡片的 frame。
3. 在 `withinItemMarkerOverlay` 中：
   - 用卡片 `maxX` 代替当前“卡片左缘入口”作为红线终点计算依据
   - 删除左缘短刻度绘制
   - 保留时间胶囊、红点和同轴横线
4. 如实际视觉冲突不明显，不修改 `EventCardView`。

## 验证标准

1. 今天存在进行中日程时，红线应从时间轴穿过当前卡片。
2. 红线不再在卡片左缘终止。
3. 红线位于卡片内容上层。
4. 非进行中场景的 marker 逻辑不变。
5. 目标测试和 macOS build 通过。
