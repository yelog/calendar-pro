# 当前时间线三轨分层重设计

**日期：** 2026-04-02

**目标：** 重新设计 Calendar Pro 下拉日程区域的当前时间表现，解决当前时间标签遮挡时间轴节点、卡片边缘打断当前时间线的问题，并保留“当前时间位于进行中日程内部”的语义。

## 问题复盘

当前实现已经解决了“当前时间落在当前日程上方”的逻辑问题，但又暴露出两个更本质的视觉问题：

1. 当前时间标签、当前时间红点、时间轴节点都挤在同一条横向通道里。
2. 当前时间线试图穿进卡片内容区，和卡片本身的边框、前导装饰、文本内容发生视觉争抢。

这说明问题不再是单个偏移值不对，而是布局结构和视觉层级不合理。

## 设计目标

1. 当前时间标签不能遮挡时间轴节点。
2. 当前时间线不能与卡片正文和卡片装饰竞争同一层级。
3. 仍然要表达“当前时间在这个卡片的哪一个纵向位置”。
4. 保持当前 popover 紧凑、清晰、可扫描的菜单栏工具风格。

## 非目标

1. 不重做为完整按小时比例展开的日视图。
2. 不改动数据拉取和筛选逻辑。
3. 不改变事件卡片的点击行为与信息结构。

## 方案对比

### 方案 A：继续微调当前 overlay 的偏移和长度

- 优点：代码改动最小
- 缺点：仍然把时间标签、时间轴节点、当前时间红点和卡片入口塞在同一条通道里，只会反复出现新的遮挡问题

### 方案 B：保留现有列结构，只把红点左右错位

- 优点：比方案 A 稍稳定
- 缺点：只是“错开”，不是“分层”；一旦字体、文案长度或卡片高度变化，问题依然会回来

### 方案 C：三轨分层重设计

- 做法：把日程行拆成独立的时间文本轨、时间轴轨、卡片内容轨；当前时间标签只占时间文本轨，当前时间红点只占时间轴轨，横向连接线只负责连接时间轴与卡片边缘，卡片内部只保留一个很短的入口刻度
- 优点：信息层级清晰，视觉冲突最少，后续扩展最稳定
- 缺点：需要重构 `EventListView` 的行布局与 marker 绘制方式

## 选定方案

采用方案 C。

## UI/UX 设计

### 一、视觉分层

整个时间线拆成三条并列视觉轨道：

1. **时间文本轨**
   - 只显示分组时间和当前时间标签
   - 当前时间使用小号红色时间胶囊，不再侵入时间轴节点区域

2. **时间轴轨**
   - 只显示垂直轨道、分组节点、当前时间红点
   - 当前时间红点位于轨道中心，不覆盖分组时间文本

3. **卡片内容轨**
   - 只显示事件/提醒事项卡片
   - 当前时间不再横穿正文，而是在卡片左缘形成一个短入口刻度，表达“当前进度已进入此卡片”

### 二、当前时间表现

#### 进行中卡片

1. 当前时间胶囊显示在时间文本轨中，与时间轴节点彻底分离。
2. 当前时间红点显示在时间轴轨中。
3. 一条细红线从时间轴轨连接到卡片左缘。
4. 在卡片左缘的对应高度绘制一个短红色入口刻度，作为时间进入卡片的视觉终点。
5. 不再让红线穿过卡片正文，避免压住标题、时间范围和前导装饰。

#### 组前 / 组后场景

1. 继续保留当前时间作为独立 marker row 的语义。
2. 但 row 的视觉也改用相同的“三轨结构”，保证整套语言一致。

### 三、交互和可读性原则

1. 当前时间是“状态指示”，不是“内容分隔线”，因此不应盖住内容。
2. 卡片内容优先级高于装饰线条，正文区域必须保持完整可读。
3. 时间轴节点和当前时间红点都属于导航语义，应该并列可见，而不是互相替代。

## 架构设计

### 数据层

继续沿用当前的 item 级 marker 模型：

- `beforeGroup`
- `withinItem(selectionIdentifier, progress)`
- `afterGroup`

不再新增新的业务状态。

### 视图层

1. 将 `timedGroupView` 从“时间标签 + 节点混合列”改为“三轨 HStack”。
2. 将原有 `timelineColumn(for:)` 拆成：
   - 时间文本轨
   - 时间轴轨
3. `withinItem` marker overlay 只负责绘制：
   - 时间胶囊
   - 红点
   - 连接线
   - 卡片左缘入口刻度
4. `beforeGroup` / `afterGroup` marker row 使用同样的三轨布局，保证一致性。

## 验证标准

1. 当前时间标签不再遮挡时间轴节点。
2. 当前时间红线不再被卡片视觉“截断”。
3. 进行中卡片仍然能通过 marker 的纵向位置表达当前时间进度。
4. 非进行中场景下，组前 / 组后 marker 的定位逻辑不变。
5. 测试和构建通过。
