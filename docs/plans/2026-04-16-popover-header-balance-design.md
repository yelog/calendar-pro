# Popover Header Balance Design

**Date:** 2026-04-16
**Status:** Approved

## Goal

调整月历弹层顶部与底部按钮布局，让年月标题恢复真正居中，同时把 `今日` 提升到日历附近，减少从事件列表底部返回今天的操作距离。

## Scope

- 将 `今日` 按钮从底部导航移动到顶部头部区域。
- 将 `休假` 按钮移到顶部左侧。
- 调整头部布局，确保年月标题相对整条头部几何中心居中，而不是被左右按钮宽度拉偏。
- 底部导航移除 `今日`，保留 `设置` 和 `退出`。

## Non-Goals

- 不修改 `休假建议` 功能的可用性判断与打开逻辑。
- 不修改月份切换、年份选择、月份选择的交互行为。
- 不改动事件列表、黄历条和月历网格布局。

## Design

### 1. Header composition

顶部头部改为三段式结构：

- 左侧：上月按钮 + `休假` 胶囊按钮
- 中间：年份按钮 + 月份按钮
- 右侧：`今日` 胶囊按钮 + 下月按钮

仅仅交换按钮位置可以改善视觉平衡，但不能从布局机制上保证标题绝对居中。因此头部整体使用 `ZStack` 叠加：

- 底层 `HStack` 负责放置左右控制组
- 上层单独放置标题按钮组

这样标题的中心点由整条头部宽度决定，不再受左右内容宽度变化影响。

### 2. Symmetry and sizing

`休假` 与 `今日` 使用同一套胶囊样式，包括：

- 相同高度
- 相同横向内边距
- 相同字体和圆角风格

两者文案在中文下都是两个字，统一样式后左右视觉重量基本一致。即使未来按钮状态颜色不同，标题仍由独立居中层保证不会被拖偏。

### 3. Footer simplification

底部导航去掉 `今日` 后，保留：

- 左侧 `设置`
- 右侧 `退出`

底部不再承担“快速返回今天”的职责，避免主要日历导航入口和辅助操作入口分散在弹层两端。

## Files Expected To Change

- `CalendarPro/Views/Popover/MonthHeaderView.swift`
- `CalendarPro/Views/Popover/CalendarPopoverView.swift`

## Risks

- 头部如果仅用 `ZStack` 强行居中但不控制可用宽度，极窄宽度下可能与左右按钮重叠；当前 popover 固定宽度足够，本次不额外引入复杂测量。
- `今日` 按钮移到顶部后，底部会更简洁，但需要确认现有键盘快捷键和无障碍标签保持不变。

## Validation

- 顶部显示顺序为：上月、`休假`、年月标题、`今日`、下月。
- 年月标题在视觉上位于头部正中，不再向左偏。
- 点击 `今日` 仍能回到当天并选中今天。
- 底部仅保留 `设置` 与 `退出`，快捷键行为不变。
