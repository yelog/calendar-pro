# 菜单栏预览与真实菜单栏一致性设计

## 背景

当前“设置 → 菜单栏”顶部的预览使用 SwiftUI `Text` 直接渲染，而真实菜单栏文本由 `StatusBarController` 绘制成 `NSImage` 后显示在 `NSStatusItem` 上。两条渲染路径不同，导致默认情况下设置页预览显示为黑色文字，但右上角真实菜单栏会根据当前系统菜单栏外观显示为白色或深色文字。

## 需求分析

- **一致性目标**：设置页预览应尽量接近用户此刻在右上角看到的真实菜单栏效果。
- **默认行为**：默认模板着色模式下，预览需要跟随当前系统浅色/深色外观，而不是沿用设置页正文文字颜色。
- **已有样式兼容**：自定义文字颜色、填充背景、加粗等能力保持现有行为，预览与真实菜单栏继续共用一套样式偏好。
- **范围控制**：本次只对齐正常显示态，不额外模拟点击高亮、按下或菜单展开中的系统高亮态。

## UI/UX 设计

“菜单栏预览”保留在原位置，但内容从普通文本替换为菜单栏风格预览条：

- 预览条使用与当前系统外观一致的浅色或深色背景。
- 预览条内部显示与真实菜单栏同源生成的文本图像，而不是单独的 SwiftUI `Text`。
- 默认模板模式下，深色菜单栏显示浅色文字，浅色菜单栏显示深色文字。
- 自定义文字颜色和填充背景时，预览继续显示对应的自定义样式。

这样用户在设置中看到的是“当前系统外观下的真实菜单栏近似镜像”，而不是受设置页正文配色影响的静态文本。

## 技术方案

- 新增共享渲染器，负责把菜单栏文本和 `MenuBarTextStyle` 转成 `NSImage`。
- 将 `StatusBarController` 中的绘制逻辑迁移到共享渲染器，避免真实菜单栏和设置页预览分叉。
- 渲染器输出保留 `isTemplate` 语义：
  - 默认模式仍生成 template image，由系统菜单栏负责着色。
  - 自定义前景色或填充背景时生成非 template image。
- 设置页预览使用 `Image(nsImage:)` 显示共享渲染器输出。
- 设置页额外根据 `colorScheme` 绘制一个简化菜单栏背景，以匹配当前系统外观。

## 风险与取舍

- 预览背景只能近似系统菜单栏材质，无法完全复制系统菜单栏真实磨砂与高亮细节；但比当前普通文本预览更一致。
- 若未来真实菜单栏文本布局再调整，只需修改共享渲染器即可，设置页预览会自动同步。
- 不引入截图、离屏窗口或私有 API，保持实现简单稳定。

## 测试要点

- 默认样式下，预览在浅色/深色模式中可读，且与真实菜单栏的默认颜色逻辑一致。
- 自定义文字颜色和填充背景时，预览与真实菜单栏继续一致。
- 共享渲染器在默认模式返回 template image，在自定义样式下返回非 template image。
- 设置页预览在文本变化和样式变化时正确刷新。
