# 设置窗口缩放与定位设计

**日期：** 2026-04-01

**目标：** 让 Calendar Pro 的设置窗口支持原生拖拽缩放，并在每次打开时定位到当前触发屏幕的中心位置，兼容多显示器和扩展屏场景。

## 现状问题

1. [`AppDelegate.swift`](/Users/yelog/workspace/swift/calendar-pro/CalendarPro/App/AppDelegate.swift) 中设置窗口用固定 `840x560` 创建，且 `styleMask` 不包含 `.resizable`，因此窗口无法调整大小。
2. [`SettingsRootView.swift`](/Users/yelog/workspace/swift/calendar-pro/CalendarPro/Views/Settings/SettingsRootView.swift) 又在视图层写死 `.frame(width: 840, height: 560)`，进一步把内容尺寸锁死。
3. 设置窗口首次打开时优先走“贴近 popover”定位，而不是居中到当前屏幕。
4. 定位计算使用 `NSScreen.main` 作为裁剪基准，这不等于“触发设置时所在的那块屏幕”，在扩展屏环境下容易把窗口放到错误位置。
5. 已经存在的设置窗口再次打开时只做 `makeKeyAndOrderFront`，不会重新对齐到当前屏幕。

## 用户目标

1. 设置窗口可以原生拖拽调整宽高。
2. 关闭后重新打开时，保留用户上次调整过的窗口尺寸。
3. 每次打开设置窗口时，窗口应出现在当前触发屏幕的中间位置。
4. 在最小可用尺寸下，设置页内容仍保持可读和可操作。

## 非目标

1. 不引入新的设置项或“恢复默认大小”按钮。
2. 不为窗口尺寸额外新增业务层持久化模型。
3. 不重构设置页信息架构，只补齐窗口层和关键页面的响应式布局。

## 方案对比

### 方案 A：原生可缩放窗口 + 当前屏幕居中 + 只持久化尺寸

- 设置窗口加入 `.resizable`
- 使用 `NSWindow` 的 autosave 机制保存尺寸
- 每次打开时以当前触发屏幕的 `visibleFrame` 做居中定位
- 位置不强制复用历史值，避免和“当前屏幕居中”目标冲突

**优点：**
- 最符合 macOS 桌面窗口习惯
- 用户可自由调整尺寸，但不会因为历史位置把窗口留在另一块屏幕
- 改动集中，风险低

**缺点：**
- 需要补一些设置页的窄宽度自适应

### 方案 B：原生可缩放窗口 + autosave 完整 frame

- 同时记住历史尺寸和位置
- 再尝试在需要时修正到当前屏幕

**优点：**
- 最大程度保留系统行为

**缺点：**
- “记住完整位置”和“每次打开都在当前屏幕中心”天然冲突
- 多屏切换下逻辑复杂

### 方案 C：只支持缩放，不做多屏定位优化

- 加 `.resizable`
- 保持当前贴近 popover 的定位逻辑

**优点：**
- 实现最小

**缺点：**
- 无法解决扩展屏场景下窗口出现在错误屏幕的问题

## 选定方案

采用方案 A。

## 设计细节

### 窗口行为

1. 默认尺寸仍为 `840x560`。
2. 最小尺寸设为 `760x520`。
3. 设置窗口允许自由拉伸，不设置最大尺寸。
4. 通过 `setFrameAutosaveName` 与 `setFrameUsingName(..., force: false)` 让系统恢复历史尺寸。
5. 恢复历史尺寸后，仍要重新居中到当前目标屏幕，而不是沿用历史位置。

### 屏幕选择规则

当前屏幕按以下优先级确定：

1. 设置入口所依附的 popover 内容窗口所在屏幕
2. 当前 key window 所在屏幕
3. 鼠标所在屏幕
4. `NSScreen.main`

### 定位规则

1. 取得目标屏幕的 `visibleFrame`
2. 用当前窗口尺寸在该区域内计算居中 origin
3. 调用 `setFrameOrigin` 放置窗口
4. 每次执行 `openSettings()` 都重新应用该规则，包括窗口已存在的情况

### 内容布局策略

1. `SettingsRootView` 不再写死宽高，只填满窗口内容区。
2. 左侧导航维持固定宽度，右侧内容区伸缩。
3. `GeneralSettingsView` 的摘要卡片网格根据可用宽度在单列/双列之间切换。
4. `MenuBarSettingsView` 的显示项行在窄宽度下从单行改成上下布局，避免控件被挤压。
5. 其他设置页继续依赖现有 `ScrollView` 处理高度收缩。

## 风险与处理

1. **自动恢复 frame 后位置错误**
   - 处理：只复用恢复出来的尺寸，再手动居中到当前屏幕。

2. **窗口缩窄后控件重叠**
   - 处理：设置最小窗口尺寸，并给 `General`、`MenuBar` 补响应式布局。

3. **重复打开设置时仍停留在旧屏幕**
   - 处理：无论窗口是否已存在，每次打开都重新居中到当前目标屏幕。

## 验证方式

1. 手动拖拽设置窗口，确认宽高可以调整。
2. 关闭并重开设置窗口，确认尺寸被保留。
3. 在扩展屏上打开设置窗口，确认出现在当前屏幕中心。
4. 将鼠标或 popover 切换到另一块屏幕再次打开，确认窗口跟随目标屏幕移动。
5. 检查 `General` 和 `MenuBar` 页在最小尺寸下没有控件遮挡或挤出。
6. 运行项目构建，确认没有编译回归。
